## 一、功能概述

用户可以向研究员发起 1V1 咨询，分为**文字咨询**和**语音咨询**两种形式。

内测阶段每位用户拥有 **1 次文字咨询 + 1 次语音咨询**权益（绑定在所选研究员的订阅权益内）。

---

## 二、页面架构

### 2.1 页面总览与导航流程

共 5 个页面，所有状态均通过页面内状态切换承接，不新增页面。

```
Page 1 研究员列表（选人）
  │ 点击研究员卡片
  ▼
Page 2 研究员主页（建立信任 + 选服务 + 查看历史记录）
  │ 点击 [文字咨询] 或 [语音咨询]
  ▼
Page 3 发起咨询（填写问题 + 选交易对 + 语音选时间）
  │ 点击 [提交]
  ▼
Page 4 咨询详情页（对话/通话 + 全部状态流转，复用为历史只读页）
  │ 结束后点击 [评价]
  ▼
Page 5 评价页（评分 + 分享）
  │ 提交评价
  ▼
Page 2（返回研究员主页，可查看咨询记录）
```

| 页面 | 名称 | 核心职责 | 状态复杂度 |
|------|------|----------|-----------|
| Page 1 | 研究员列表 | 选人 | 低（静态列表） |
| Page 2 | 研究员主页 | 建立信任 + 服务入口 + 历史记录 | 低（按钮状态随权益变化） |
| Page 3 | 发起咨询 | 填写问题 + 选咨询方式 | 低（表单页） |
| Page 4 | 咨询详情页 | 对话/通话 + 全部状态流转 | **高**（文字 5 状态 + 语音 6 状态） |
| Page 5 | 评价页 | 评分 + 引导 | 低（表单页） |

#### 正常流程跳转

**文字咨询：**

| 步骤 | 用户操作 | 从 | 到 |
|------|---------|-----|-----|
| 1 | 点击研究员卡片 | Page 1 列表 | Page 2 主页 |
| 2 | 点击 [文字咨询] | Page 2 | Page 3 发起咨询 |
| 3 | 填写问题，点击 [提交] | Page 3 | Page 4（状态 1：等待接单） |
| 4 | 研究员接单 | Page 4 状态 1 | Page 4（状态 2：对话中） |
| 5 | 用户/研究员/系统结束 | Page 4 状态 2 | Page 4（状态 4：已结束） |
| 6 | 点击 [评价研究员] | Page 4 状态 4 | Page 5 评价页 |
| 7 | 提交评价 / 跳过 | Page 5 | Page 2 主页（底部可见本次记录） |

**语音咨询：**

| 步骤 | 用户操作 | 从 | 到 |
|------|---------|-----|-----|
| 1-2 | 同文字咨询 | Page 1 | Page 2 → Page 3 |
| 3 | 填写问题 + 选时间，点击 [提交预约] | Page 3 | Page 4（状态 A：等待确认） |
| 4 | 研究员确认 | Page 4 状态 A | Page 4（状态 B：已确认） |
| 5 | 预约时间到达 | Page 4 状态 B | Page 4（状态 C：可拨打） |
| 6 | 点击 [开始通话]，研究员接听 | Page 4 状态 C | Page 4（状态 D：通话中） |
| 7 | 任一方挂断 | Page 4 状态 D | Page 4（状态 E：通话结束） |
| 8 | 点击 [评价研究员] | Page 4 状态 E | Page 5 → Page 2 |

> 异常中断的跳转逻辑详见第三章 3.4 和第四章 4.4。

|------|------|------|

| 系统通用 | 平台统一配置，所有研究员共用 | [趋势分析] [风险预警] [入场点位] |

> 研究员预设标签排在前面、系统通用排在后面。若研究员未配置预设问题，则仅显示系统通用标签。

#### 交互规则

| 交互 | 说明 |
|------|------|
| 问题为空 | 提交按钮置灰不可点 |
| 点击快捷标签 | 文字自动填入输入框，用户可继续编辑修改 |
| 语音 — 立即通话 | 研究员在线时可选；离线时该选项禁用并置灰，自动选中"预约时间" |
| 语音 — 预约时间 | 选择日期 + 时段，仅展示研究员可用时段 |
| 点击 [发起咨询] | 扣减 1 次对应权益 → 跳转 Page 4 |
| 点击 [←] 返回 | → 返回 Page 2，已填内容不丢失（返回后再进来可恢复） |
| 异常回填 | 从 Page 4 异常中断跳回时（换研究员后重选），问题内容自动回填到新研究员的 Page 3 |

### 2.5 Page 4：咨询详情页

#### 功能定位

整个产品**状态最复杂的页面**。文字咨询和语音咨询共用同一页面结构，通过顶部状态条区分阶段。同时复用为历史咨询记录的只读查看页。

#### 通用页面结构

```
┌──────────────────────────────────┐
│  ← Vigo · [咨询类型]             │ ← 导航栏
├──────────────────────────────────┤

#### 状态条规范

| 底色 | 含义 | 使用场景 | 示例文案 |

> Page 4 的**分状态详细设计**见后续第三章（文字咨询）和第四章（语音咨询）。

#### 跳转规则总结

以下规则适用于 Page 4 所有异常中断场景的跳转：

| 规则 | 说明 |
|------|------|

| [重新预约] 统一跳 Page 3 | 同一研究员，问题回填，重选时间 |

## 三、文字咨询（Page 4 状态详解）

以下是 Page 4 在文字咨询场景下的完整状态拆解。

### 3.1 核心规则

| 规则 | 说明 |
|------|------|
| 对话形式 | 聊天窗口，双方以消息气泡来回对话 |
| 有效期 | 研究员接单起 **24 小时** |
| 消息条数 | **不限**（设一个防刷上限如 30 条，不在 UI 展示） |
| 用户结束 | 随时可点"结束咨询" |
| 研究员结束 | **必须已回复 >= 3 次**，否则按钮置灰不可点 |
| 系统结束 | 24 小时到期自动关闭 |

> **"回复"的定义**：一个来回算一次。用户发消息（不管几条）→ 研究员回复（不管几条）= 研究员回复 1 次。

### 3.2 状态流转概览

**正常流程：**

| 步骤 | 状态 | 下一步 |
|------|------|--------|
| 1 | 用户提交问题 | → 等待接单 |
| 2 | 等待接单 | → 研究员接单 → 对话中 |
| 3 | 对话中 ⇄ 等待回复 | → 24h 到期 / 任一方结束 → 已结束 |
| 4 | 已结束 | → 评价（Page 5） |

> 实时通话（若支持）：跳过 A、B，直接进入可拨打（C）→ 通话中（D）→ 通话结束（E）

**异常分支：** 详见 [3.4 中断处理与异常跳转](#34-中断处理与异常跳转)。

### 3.3 页面状态详解

Page 4 对话页通过**顶部状态条**切换状态，共 5 个状态：

#### 状态 1：等待接单

```
┌──────────────────────────────────┐
│  ← Vigo · 文字咨询               │
├──────────────────────────────────┤
│  ┌─ 你的提问 ──────────────────┐ │
│  │ BTC/USDT                    │ │
│  │ 当前适合入场吗？              │ │
│  └─────────────────────────────┘ │
│                                  │
│       ⏳ 等待研究员接单            │
│       通常 5 分钟内响应            │
│                                  │
│          [取消咨询]               │ ← 用户可取消
├──────────────────────────────────┤
│  输入框锁定（灰色）               │
│  "研究员接单后即可对话"            │
└──────────────────────────────────┘
```

- 输入框锁定，防止用户在接单前刷屏
- 显示预期响应时长，降低焦虑
- [取消咨询] 点击后弹出二次确认「确认取消？权益将自动退还」→ 确认后权益退还，返回 Page 2

#### 状态 2：对话进行中

```
┌──────────────────────────────────┐
│  ← Vigo · 文字咨询               │
│  [蓝底] 对话中 · 剩余 18:32:00  [结束咨询] │
├──────────────────────────────────┤
│  [用户消息气泡]          12:01    │
│            [研究员回复气泡] 12:03  │
│  [用户追问]              12:15    │
│            [研究员回复]    12:18   │
├──────────────────────────────────┤
│  [📎] [输入框..........] [发送]   │
└──────────────────────────────────┘
```

- 顶部状态条显示剩余时间倒计时，右侧放 [结束咨询] 按钮
- 用户点击 [结束咨询] → 弹出二次确认「确认结束咨询？结束后不可继续发送消息」→ 确认后进入状态 4（已结束）
- 支持发送文字和图片/截图（K线图等）
- 研究员侧：结束按钮旁显示 `[结束咨询（还需回复 X 次）]`，不足 3 次时置灰

#### 状态 3：等待研究员回复

```
┌──────────────────────────────────┐

├──────────────────────────────────┤

├──────────────────────────────────┤

```

- 用户可继续发送消息（补充信息）
- 研究员回复后，用户可通过 Page 1 顶部横幅快捷返回对话

#### 状态 4：已结束

```
┌──────────────────────────────────┐

├──────────────────────────────────┤
│  ... 完整对话记录（只读）...       │

- 输入框消失，对话变为只读
- 对话记录永久可查
- 评价按钮引导进入 Page 5

#### 状态 5：异常（拒绝/超时）

- "换一位研究员"跳回 Page 1，问题内容保留不用重新填写

### 3.4 中断处理与异常跳转

所有中断通过 Page 4 顶部状态条处理，不新增页面。以下为**唯一权威定义**，包含状态条颜色、文案、用户可操作项和跳转目标。

| 场景 | 状态条 | 用户可操作 | 跳转目标 | 权益 |

| 对话中 → 研究员 48h 未回复 | [红底] 研究员暂未回复 | [结束并退还权益] | Page 4 已结束（对话记录保留可查） | 退还 |

| 对话中 → 24 小时到期 | [绿底] 已结束 | [评价] | Page 5 | 不退 |

**要点：**
- 研究员不能在回复不足 3 次时单方面结束
- 研究员主动结束后给用户 2 小时缓冲窗口可继续发消息
- 研究员 48 小时不回复时，由用户自行选择是退还权益还是继续等
- 研究员拒绝/超时使用中性文案（"暂时无法服务""暂未响应"），不使用负面措辞

## 四、语音咨询（Page 4 状态详解）

以下是 Page 4 在语音咨询场景下的完整状态拆解。语音咨询复用 Page 4 的通用结构，增加通话面板组件。

### 4.1 核心规则

| 规则 | 说明 |
|------|------|
| 发起方式 | 预约制（用户选时间段）或实时发起 |
| 通话时长 | 待定（建议 30 分钟，需确认） |
| 通话后追问 | 不单独设追问轮次，想追问另开文字咨询 |
| 5 分钟分水岭 | 通话不足 5 分钟视为未完成服务 |

### 4.2 状态流转概览

**正常流程（预约制）：**

| 步骤 | 状态 | 下一步 |
|------|------|--------|
| A | 用户提交预约 | → 等待确认 |
| B | 研究员确认 | → 等待通话时间 |
| C | 通话时间到达 | → 可拨打 |
| D | 用户拨打，研究员接听 | → 通话中 |
| E | 任一方挂断 | → 通话结束 → 评价（Page 5） |

> 实时通话（若支持）：跳过 A、B，直接进入可拨打（C）→ 通话中（D）→ 通话结束（E）

**异常分支：** 详见 [4.4 中断处理与异常跳转](#44-中断处理与异常跳转)。

### 4.3 页面状态详解

语音咨询复用 Page 4，通过状态切换区分阶段，共 6 个状态：

#### 状态 A：预约已提交，等待确认

## 六、核心设计原则

| 原则 | 具体规则 |
|------|----------|
| 谁的过错谁承担 | 研究员导致的中断 → 退权益；用户导致的 → 不退 |
| 永远给出路 | 任何中断至少提供 2 个选项，不能是死胡同 |
| 5 分钟分水岭 | 语音通话不足 5 分钟视为未完成服务 |
| 不让用户主动维权 | 超时/拒绝的权益退还必须是自动的 |
| 中性文案 | 研究员拒绝/不回复时用"暂时无法""暂未响应"，不用负面措辞 |
| 保留用户输入 | 任何中断都不清除用户已填写的问题内容，方便复用 |
| 页面状态承接 | 所有状态通过 Page 4 顶部状态条切换，不新增页面 |

## 七、权益处理汇总

| 场景 | 权益 |
|------|------|
| 服务未开始（未接单/未确认/被拒绝/超时） | 自动退还 |

| 语音通话 >= 5 分钟 | 不退 |

## 附录：Demo 原型方案（备忘）

### 思路

- **用户侧操作**做成真实可点击的 UI（按钮、表单、发消息）
- **研究员侧操作**通过浮动控制面板模拟（Accept / Reply / Reject 等）
- 状态不手动切换，由用户点击 + 研究员模拟操作自然触发流转
- Demo 界面上的产品文案统一用**英文**

### 布局

桌面端：左侧手机模拟框（375×812），右侧研究员控制面板。移动端：控制面板收到底部抽屉。

### 控制面板按钮

文字咨询：`Accept` / `Reply` / `End Session`（>=3 次回复可点）/ `Reject` / `Timeout 15m` / `No Reply 48h`

语音咨询：`Confirm` / `Time's Up` / `Pick Up` / `Hang Up` / `Reject` / `Timeout 4h` / `No Answer ×3` / `Disconnect`

面板只显示当前状态下可用的按钮，其余隐藏或置灰。

### 模拟行为

- 24h 倒计时加速（1 真实秒 = 1 模拟分钟）
- 通话计时真实计时
- 重连 60s 倒计时归零自动结束
- 权益本地 state 追踪，提交扣减，异常退还
- 异常中断时问题内容保留，换人后自动回填

### 技术栈

Next.js + Tailwind，约 12 个组件文件，预估 800-1000 行代码。部署 Vercel。

### 演示走查顺序

1. 正常文字咨询：P1→P2→P3A→P4(waiting)→[Accept]→聊天→[Reply]×3→用户结束→P5→P2
2. 异常-研究员拒绝：P4(waiting)→[Reject]→用户点 Switch→P1
3. 异常-48h 不回复：聊天中→[No Reply 48h]→用户点 End & Refund
4. 正常语音咨询：P3B→P4(pending)→[Confirm]→[Time's Up]→Start Call→[Pick Up]→展开/收缩→[Hang Up]→P5
5. 异常-语音未接通：响铃→[No Answer ×3]→用户点 Try Text Chat
6. 异常-通话断网：通话中→[Disconnect]→重连 60s→自动结束
